# 知知 — 功能清单

## 产品主轴

核心定位：管理者发令，AI 并行推进，关键节点请示，结果回到会话。

交互闭环：`输入 → 理解 → 执行 → 确认/结果`。

多会话模型：每个会话是独立上下文容器，内含多条指令和任务。用户通过新建会话开启新主题，实现多任务并行。

Local-first 架构：数据先存本地 SQLite，UI 从本地读取秒开，后台定期同步到服务端。离线可用。

关联文档：[prd.md](prd.md)、[voice-interaction-loop.md](voice-interaction-loop.md)、[technical-architecture.md](technical-architecture.md)、[design-guidelines.md](design-guidelines.md)。

## 一、必须有（核心闭环）

围绕"发令 → 推进 → 拍板 → 看结果"这条主线，以下功能缺一不可。

### 1.1 会话管理

| 功能 | 说明 |
|------|------|
| 会话列表 | 首页展示所有会话，按最近活跃排序 |
| 新建会话 | 一键创建新会话，开启新主题 |
| 会话详情 | 对话视图，展示用户指令气泡 + AI 响应卡片 |
| 删除会话 | 左滑删除不需要的会话 |
| 会话标题 | AI 自动生成或用户手动命名 |
| 本地存储 | 数据先存本地 SQLite，离线可用 |
| 定期同步 | 后台定期同步到服务端，网络恢复后自动补传 |

### 1.2 输入层

| 功能 | 说明 |
|------|------|
| 语音输入 | 按住说话，松开即发出，上滑取消 |
| 文字输入 | 打字发送指令 |
| 图片输入 | 拍照、截图、相册选择 |
| 混合输入 | 语音 + 图片、文字 + 图片同时发送 |
| 统一输入口 | 只有一个入口，不需要先选类型或 Bot |

### 1.3 多任务并行

| 功能 | 说明 |
|------|------|
| 多任务同时执行 | 多条指令可同时在后台推进 |
| 局部阻塞 | 某任务待确认不影响其他任务继续执行 |
| 首页战况 | 打开首页即可看到待拍板、执行中、异常 |
| 随时插入新任务 | 在任何页面都可以立刻发新指令 |

### 1.4 任务流

| 功能 | 说明 |
|------|------|
| 按状态展示 | 待确认、执行中、已完成、失败 |
| 状态卡片 | 每条任务显示原始指令、当前状态、进展摘要 |
| 分层排列 | 待确认置顶，执行中其次，已完成沉底 |
| 异常突出 | 失败、超时、需介入的任务视觉强提醒 |

### 1.5 任务详情

| 功能 | 说明 |
|------|------|
| 理解展示 | "我理解的是……" |
| 执行计划 | "我准备怎么做……" |
| 执行进展 | 当前步骤、阶段进度 |
| 最终结果 | 结果卡片 + 可操作项 |

### 1.6 确认机制

| 功能 | 说明 |
|------|------|
| 风险分级 | 低风险直接执行，中风险建议确认，高风险必须确认 |
| 确认卡片 | 展示将执行什么、影响谁、为什么需要拍板 |
| 确认/取消 | 一键确认或取消 |
| 执行中追加确认 | 执行过程中发现新风险，暂停并再次请示 |

### 1.7 追问/补充/纠偏

| 功能 | 说明 |
|------|------|
| 追问 | "这个再详细说说" |
| 补充 | "还有一个条件忘了说" |
| 纠偏 | "不是这个意思""重点不对" |
| 叫停 | "先别做了" |
| 改判 | "这个不对，改成这样" |

### 1.8 结果卡片

| 功能 | 说明 |
|------|------|
| 结构化结果 | 清单、摘要、表格、文档 |
| 语音播报摘要 | 可选 TTS 播报结果要点 |
| 可操作结果 | 结果本身可以继续处理 |

### 1.9 失败与异常

| 功能 | 说明 |
|------|------|
| 失败定位 | 告知哪一步失败 |
| 失败原因 | 信息不足、权限不足、工具失败、目标不清 |
| 建议补救 | 提供可行的下一步建议 |
| 重试入口 | 支持一键重试 |

### 1.10 叫停能力

| 功能 | 说明 |
|------|------|
| 暂停执行 | 执行中任务可随时暂停 |
| 取消任务 | 不想继续的任务可以直接取消 |
| 撤回确认 | 确认后如还未产生外部影响，允许短时间撤回 |

## 二、应该有（增强体验）

这些功能会让产品从"能用"走向"好用"，直接影响留存。

### 2.1 任务归属识别

| 功能 | 说明 |
|------|------|
| 自动判断 | 新输入是新任务，还是旧任务的追问/补充 |
| 上下文归属 | "刚才那个"能正确关联到对应任务 |
| 不确定时反问 | 置信度低时主动问用户"这句话是关于哪件事" |

### 2.2 优先级与调度

| 功能 | 说明 |
|------|------|
| 优先级调整 | "这个先做""这个放后面""这个今天一定要出结果" |
| 系统自动排序 | 基于待确认、异常、超时、用户关注度排序 |
| 紧急插队 | 新指令如果紧急，可以优先调度资源 |

### 2.3 批量拍板

| 功能 | 说明 |
|------|------|
| 合并确认 | 多条类似低风险确认项合并成一条 |
| 一键全部确认 | 对聚合后的确认项支持批量操作 |
| 确认压缩 | 系统自动识别可合并的确认请求 |

### 2.4 任务依赖

| 功能 | 说明 |
|------|------|
| 串行依赖 | "A 做完再开始 B" |
| 汇总依赖 | "这几件事都做完后帮我汇总" |
| 自动触发 | 前置任务完成后自动启动后续任务 |

### 2.5 持续关注

| 功能 | 说明 |
|------|------|
| 盯办模式 | "帮我盯着华东区异常" |
| 周期巡检 | 每天、每周自动检查某类事项 |
| 静默推进 | 默认不打扰，只有关键节点冒出来 |
| 超时升级 | 超过预期时长自动升级为异常 |

### 2.6 阶段汇报

| 功能 | 说明 |
|------|------|
| 里程碑汇报 | 关键步骤完成时汇报 |
| 异常汇报 | 发现问题时立即汇报 |
| 打扰节制 | 普通进展不打扰，只更新任务流 |

### 2.7 规则记忆

| 功能 | 说明 |
|------|------|
| 规则识别 | 从对话中识别规则类意图 |
| 规则持久化 | "涉及预算先问我""类似事项直接做" |
| 规则执行 | 后续任务自动遵守已设定规则 |
| 规则冲突 | 新旧规则冲突时主动问用户以哪个为准 |

### 2.8 历史复用

| 功能 | 说明 |
|------|------|
| 指代消解 | "上次那个事""跟之前一样" |
| 范式复用 | "按上次那个风格来" |
| 任务统计 | "这个月我一共交代了多少件事" |

### 2.9 晨间简报 / 晚间复盘

| 功能 | 说明 |
|------|------|
| 晨间简报 | 今天最该你盯的 3—5 件事 |
| 晚间复盘 | 今天交代了什么、完成了什么、还卡着什么 |
| 阶段汇总 | 本周/本月整体推进情况 |

### 2.10 结果后续动作

| 功能 | 说明 |
|------|------|
| 继续处理 | 基于结果发起下一步指令 |
| 转文档 | 一键转成简报、邮件草稿、会议纪要 |
| 设提醒 | "明天再提醒我看这个" |
| 转任务 | 把结果中的事项结构化落地为任务 |
| 结果摘要层级 | 30 秒摘要、3 分钟摘要、完整详情三档切换 |

## 三、后续增强（产品壁垒）

这些功能会形成竞争壁垒，让用户越用越离不开。

### 3.1 外部通知与协作

| 功能 | 说明 |
|------|------|
| 发 IM 消息 | 代发消息给团队成员 |
| 发邮件 | 生成正式邮件并发送 |
| 同步日历 | 创建、修改日程 |
| 代约会议 | 需要同步时自动建议建会 |
| 回收反馈 | 下发任务后拉回执行反馈 |

### 3.2 数据/BI 接入

| 功能 | 说明 |
|------|------|
| 查询经营数据 | 接入数据库或 BI 系统查询 |
| 自动发现异常 | 基于数据波动主动发现异常并汇报 |
| 数据对比 | 同一主题能看"上次 vs 这次"的差异 |

### 3.3 协作闭环

| 功能 | 说明 |
|------|------|
| 下发给人 | 一条指令分发给多个角色 |
| 汇总进度 | 多方进度自动汇总 |
| 自动催办 | 根据任务状态判断是否该催，不是简单定时提醒 |

### 3.4 管理风格学习

| 功能 | 说明 |
|------|------|
| 输出风格 | 学会用户偏好简报型、清单型还是先结论后细节 |
| 确认习惯 | 学会哪些事用户总是确认、哪些总是取消 |
| 关注重点 | 学会用户最常关注的领域和维度 |
| 术语记忆 | 记住内部黑话、项目代号、团队叫法 |

### 3.5 关系记忆

| 功能 | 说明 |
|------|------|
| 人物关系 | 知道小王负责什么、哪个客户归谁 |
| 项目关系 | 哪个项目属于哪条业务线 |
| 区域关系 | 华东区包含哪些门店 |

### 3.6 冲突检测

| 功能 | 说明 |
|------|------|
| 目标冲突 | "既要压预算又要提服务响应" |
| 规则冲突 | 新规则和旧规则矛盾 |
| 重复指令 | 已经在跑的事被重复交代时提示 |

### 3.7 打扰预算

| 功能 | 说明 |
|------|------|
| 打扰上限 | 系统每天最多主动打扰 N 次 |
| 全局暂停 | "我在开会，非紧急先别打扰我" |
| 智能聚合 | 多条非紧急通知合并为一条摘要 |

### 3.8 完整审计

| 功能 | 说明 |
|------|------|
| 操作留痕 | 所有 AI 执行和用户确认全部留痕 |
| 追责链路 | 回看"这件事是谁在什么时候因为什么停住了" |
| 按角色限权 | 不同角色可调用不同工具 |
| 敏感操作双确认 | 高影响动作需二次确认 |

### 3.9 结果可信度

| 功能 | 说明 |
|------|------|
| 可信度标注 | 标注结果基于完整数据、部分信息还是推测 |
| 执行预估 | 开始前告知"大约需要多长时间" |
| 建议下一步 | 执行完后自动推荐最自然的后续动作 |
| 结果模板化 | 把满意的输出固化为以后默认风格 |

## 四、不建议过早做

| 不做 | 原因 |
|------|------|
| 复杂仪表盘 | 会把产品拉回传统 BI 工具 |
| 功能菜单式导航 | 违背"只有一个入口"的原则 |
| Bot 市场 | 分散核心体验，变成平台型产品 |
| 手动配置很多规则的设置中心 | 规则应通过对话设定，不应进设置页 |
| 像 IM 一样的重聊天体验 | 这是指挥台，不是聊天软件 |

## 五、推荐分期

### V0.1 — 验证核心闭环

目标：多会话 + local-first + 发指令 → 理解 → 执行 → 结果。

| 功能 | 来源 |
|------|------|
| 会话列表 + 新建会话 + 会话详情 | 1.1 会话管理 |
| 本地 SQLite 存储 + 定期同步 | 1.1 会话管理 |
| 语音输入 + 文字输入 | 1.2 输入层 |
| 单任务基础闭环 | 1.4 任务流 |
| 结果卡片 | 1.8 结果卡片 |
| 失败处理 | 1.9 失败与异常 |

验证点：新建会话 → 发指令 → 理解 → 执行 → 结果显示在对话流中。离线时数据不丢失。

### V0.2 — 多任务 + 确认 + 多模态

目标：管理者可以在多个会话中连续发指令，图片也能发，关键操作需确认。

| 功能 | 来源 |
|------|------|
| 多会话并行（多个会话同时有任务在跑） | 1.3 多任务并行 |
| 图片输入 + 混合输入 | 1.2 输入层 |
| 任务详情页 | 1.5 任务详情 |
| 确认机制 | 1.6 确认机制 |
| 追问/补充/纠偏 | 1.7 追问/补充/纠偏 |
| 叫停能力 | 1.10 叫停能力 |
| 会话内上下文自动关联 | 2.1 任务归属识别 |

验证点：在 3 个不同会话中各发指令 → 并行推进 → 其中一条需确认 → 其余不受阻塞。

### V0.3 — 记忆 + 持续推进

目标：产品不只响应指令，还能主动推进和提醒，越用越懂你。

| 功能 | 来源 |
|------|------|
| 规则记忆 | 2.7 规则记忆 |
| 历史复用 | 2.8 历史复用 |
| 持续关注 | 2.5 持续关注 |
| 阶段汇报 | 2.6 阶段汇报 |
| 晨间简报/晚间复盘 | 2.9 晨间简报/晚间复盘 |
| 优先级调整 | 2.2 优先级与调度 |
| 任务依赖 | 2.4 任务依赖 |
| 批量拍板 | 2.3 批量拍板 |
| 结果后续动作 | 2.10 结果后续动作 |

验证点："帮我盯着"类指令持续运行 → 发现异常主动汇报 → 晨间简报可用。

### V0.4 — 外部工具 + 协作

目标：管理者的指令可以产生真实的外部影响。

| 功能 | 来源 |
|------|------|
| 外部通知（IM、邮件、日历） | 3.1 外部通知与协作 |
| 协作闭环 | 3.3 协作闭环 |
| 完整审计 | 3.8 完整审计 |
| 打扰预算 | 3.7 打扰预算 |

验证点：一条指令 → 自动发消息给负责人 → 回收反馈 → 汇总。

### V1.0 — 完整的知知

目标：成为管理者离不开的参谋团队。

| 功能 | 来源 |
|------|------|
| 管理风格学习 | 3.4 管理风格学习 |
| 关系记忆 | 3.5 关系记忆 |
| 冲突检测 | 3.6 冲突检测 |

验证点：用了一个月后，系统的理解准确率、推荐准确率明显提升。

## 六、功能取舍的判断标准

每次评估一个功能是否要做，问四个问题：

1. 这个功能是否让管理者更少操作？
2. 这个功能是否让 AI 更能持续推进？
3. 这个功能是否让拍板更精准而不是更频繁？
4. 这个功能是否会把产品做回传统任务管理或聊天工具？

前三个答"是"就做，第四个答"是"就不做。
