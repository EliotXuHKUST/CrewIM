# 知知 — 设计规范

## 一、设计哲学

**界面退到最后，内容站到最前。**

这不是一个让人"逛"的产品，而是一个让人"用完就走"的产品。管理者打开它只为两件事：发指令、看结果。设计的一切决策都服务于：减少视觉噪音，让管理者的注意力集中在内容本身。

### 关键词

安静、克制、高效、信任感。

### 设计原则

1. **少即是多**：能用一个元素解决的，不用两个。能不放的，就不放。
2. **内容即界面**：文字、状态、结果本身就是界面，不需要额外装饰来"说明"它们。
3. **状态清晰**：管理者扫一眼就知道哪些事在跑、哪些等他拍板、哪些已完成。
4. **操作直觉**：不需要学习成本。按住说话、点击确认、上滑取消，都是本能操作。
5. **克制动效**：动画只服务于状态切换和操作反馈，不服务于"好看"。

## 二、色彩体系

### 主色调

以中性色为主基调，整体偏暖白 + 深灰文字。90% 中性色 + 10% 语义色。

```
背景色
  主背景        #F7F7F5（暖灰白，柔和不刺眼）
  次级背景      #F1F0EC（用于输入底色、辅助面）
  卡片背景      #FFFFFF（纯白，与主背景形成微弱层级差）
  深色模式背景   #0E0E0D（接近纯黑）
  深色模式次级   #1F1F1D
  深色模式卡片   #171716

文字色
  主文字        #18181B（高可读性）
  次要文字      #6B7280（中灰，辅助信息、时间戳、状态描述）
  占位文字      #9CA3AF（浅灰，输入框 placeholder）
  深色模式主文字 #F2F1EE
  深色模式次要   #A1A1AA
```

### 功能色

只在需要传递明确含义时使用彩色，且仅用于小面积元素（状态点、按钮、标签）。

```
强调色（确认/主操作）   #2457E6（蓝，冷静、专业、可信赖）
成功色（已完成）        #0F8A5F（绿，沉稳而非鲜亮）
警告色（待确认/风险）    #B86A19（琥珀，引起注意但不刺眼）
错误色（失败）          #C53B2C（红，仅用于失败状态）
执行中                  #2457E6（蓝，与强调色统一）
深色模式强调色          #4C7DFF
```

### 用色规则

- 背景永远是中性色，不用彩色背景
- 彩色只出现在：状态指示点（6px 圆点）、按钮、极少数标签
- 卡片之间靠间距和背景微色差区分，不靠边框或阴影
- 深色模式中，功能色亮度适当降低，避免在暗背景上过于刺眼

## 三、字体体系

### 字体选择

```
中文    SF Pro（iOS）/ 思源黑体 HarmonyOS Sans（Android）
英文    SF Pro（iOS）/ Roboto（Android）
等宽    SF Mono / JetBrains Mono（代码/数据展示场景）
```

Flutter 中使用系统默认字体即可，不额外引入自定义字体，保证加载速度和系统一致性。

### 字号层级

```
超大标题    28px  semibold   仅用于 Onboarding 引导大标题
大标题      22px  semibold   页面级标题（极少使用，产品内几乎无页面标题）
卡片标题    17px  medium     任务卡片中的指令文本/理解摘要
正文        15px  regular    结果正文、详情描述、对话内容
辅助文字    13px  regular    时间戳、状态描述、次要信息
最小文字    11px  regular    极少使用，仅在标签内
```

### 行高

```
标题类    1.3x
正文类    1.6x（保证中文可读性）
辅助类    1.4x
```

### 字重规则

- 只用 regular (400) 和 medium (500) 两种字重
- semibold (600) 仅用于超大标题和大标题
- 不用 bold (700)，避免视觉过重
- 不用 light (300)，避免可读性差

## 四、间距与布局

### 间距系统

基于 4px 网格，所有间距为 4 的倍数。

```
最小间距     4px    元素内极小间隔（图标与文字）
紧凑间距     8px    同组元素间（状态点与状态文字）
标准间距    12px    卡片内部元素间
宽松间距    16px    卡片内边距
区域间距    20px    卡片之间
大区域间距  32px    区域/模块之间
```

### 卡片规范

```
内边距          16px 水平，16px 垂直
卡片间距        12px
圆角            12px
背景            #FFFFFF（浅色模式）/ #1A1A1A（深色模式）
阴影            无（靠背景色差区分层级）
边框            无
最大宽度        100%（移动端全宽，左右留 16px 屏幕边距）
```

### 屏幕边距

```
水平边距        16px（左右各 16px）
安全区域        遵循 iOS Safe Area / Android System Bars
```

### 布局原则

- 单列布局，不做多列
- 垂直滚动，不做水平滑动卡片
- 卡片占满宽度，不做浮动卡片
- 底部输入区固定，任务流区域可滚动

## 五、组件规范

### 5.1 按住说话按钮

产品最核心的交互元素。

```
默认态
  大小          64 x 64 px
  形状          正圆
  背景          #2563EB（强调色）
  图标          麦克风线性图标，白色，24px
  位置          屏幕底部居中，距底部安全区 24px
  周围          无其他元素干扰，上方留 16px 到输入栏

按下态
  大小          动画放大到 80 x 80 px（200ms ease-out）
  背景          #1D4ED8（加深）
  外圈          脉冲扩散动画（2 圈，透明度从 30% 渐隐到 0%）
  反馈          设备振动（轻触觉反馈 HapticFeedback.lightImpact）
  附加          按钮上方出现录音时长文字（13px，灰色）
  声波          按钮周围 3 条声波线，随音量振幅变化

上滑取消态
  触发距离      上滑 > 80px
  按钮颜色      变为 #DC2626（红色）
  提示文字      "松开取消"，按钮正上方，13px 红色

松开后
  大小          动画缩回 64px（150ms ease-in）
  状态          变为 loading（旋转指示器替代麦克风图标）
  持续          直到收到 task_created 事件后恢复默认态
```

## 六、页面规范

### 6.1 主屏（CommandScreen）

首页是"发令台 + 战况"，不是聊天历史。管理者打开即看到：哪些事卡住了、哪些事在跑。

```
结构（从上到下）
  ├── 状态栏（系统）
  ├── 顶部区域（最小化）
  │     极简 logo 或产品名，小字，左对齐
  │     右侧：连接状态指示（绿点 = 在线，灰点 = 离线）
  │     高度：44px
  │     与待拍板区之间：16px 间距
  ├── 待拍板区（条件显示，最多 3 条）
  │     只在有待确认任务时出现
  │     琥珀色左侧标记条（4px 宽 #D97706）
  │     卡片间距 8px（比普通卡片更紧凑）
  │     与任务流之间：20px 间距 + 1px 分隔线 #E5E7EB
  ├── 任务流区域（可滚动，占据主体）
  │     按分层排列：执行中 → 持续关注 → 已完成
  │     同层内按时间倒序
  │     卡片间距 12px
  │     执行中任务很多时自动聚合为摘要卡
  │     无任务时显示空态
  ├── 输入栏
  │     文字输入 + 图片附件
  └── 按住说话按钮
        距底部安全区 24px
```

```
│▌  │   取消    │  │   确认    │    │
```

### 6.2 任务详情（TaskDetailScreen）

详情页固定分为 4 段，展示管理者看得懂的执行过程，不暴露 Agent 内部细节。

```
结构
  ├── 顶部
  │     左侧：返回箭头（←）
  │     右侧：活跃任务指示（如"还有 2 条待确认"），13px #D97706，可点击切换
  ├── 第 1 段：我理解的是
  │     原始指令（17px medium）+ 图片缩略图（可点击放大）
  │     "我理解你需要：" 13px #6B7280
  │     理解描述 15px regular
  ├── 分隔线（1px #E5E7EB）
  ├── 第 2 段：我准备怎么做
  │     执行计划步骤列表
  │     每步一行，简短描述 15px regular
  ├── 分隔线
  ├── 第 3 段：执行进展
  │     每个子任务一行
  │     ✓ 完成 / ● 执行中 / ✗ 失败 / ○ 待执行
  │     描述 15px regular
  ├── 分隔线
  ├── 第 4 段：最终结果 / 需要你决定
  │     结果卡片（见 5.4）+ 后续操作（继续处理/转文档/设提醒）
  │     或确认卡片（影响 + 原因 + 确认/取消）
```

## 七、动效规范

### 原则

- 动效只用于状态切换和操作反馈
- 时长控制在 150-300ms
- 曲线统一用 ease-out（快进慢出）
- 不做入场动画、不做弹跳、不做 3D 翻转

### 具体动效

```
按住说话按下      200ms ease-out   按钮放大 64→80px + 脉冲出现
按住说话松开      150ms ease-in    按钮缩回 80→64px
卡片出现          200ms ease-out   从下方 offset 20px 滑入 + 透明度 0→1
卡片状态切换      250ms ease-out   内容淡入淡出（crossfade）
页面跳转          300ms ease-out   从右侧滑入（iOS 风格）
下拉刷新          弹性曲线         跟随手指 + 松开回弹
确认按钮点击      100ms           按钮轻微缩小再恢复（scale 0.95→1）
```

### 禁止的动效

- 全屏渐变背景动画
- 卡片翻转
- 弹跳/回弹过度效果
- 持续循环动画（除录音波形和执行中脉冲点外）
- 路径动画
- 粒子效果

## 八、图标规范

### 风格

- 线性图标（outline），不用填充图标
- 线宽 1.5px，统一
- 尺寸 24px（默认）/ 20px（紧凑场景）/ 16px（极小场景）
- 颜色跟随文字色（主文字色或次要文字色）

## 九、深色模式

### 色彩映射

```
浅色                    深色
#FAFAFA（主背景）    →  #0F0F0F
#FFFFFF（卡片背景）  →  #1A1A1A
#1A1A1A（主文字）    →  #F0F0F0
#6B7280（次要文字）  →  #9CA3AF
#F3F4F6（输入框背景）→  #262626
#E5E7EB（分隔线）    →  #2D2D2D
#2563EB（强调色）    →  #3B82F6（微调亮度）
```

### 原则

- 深色模式不是简单反转，而是独立调配
- 保持同样的信息层级和视觉重量
- 功能色在深色背景上适当提亮
- 阴影在深色模式下完全去除

## 十、不要做的事

- 不要加 BottomNavigationBar / TabBar
- 不要加侧边栏抽屉（Drawer）
- 不要加浮动操作按钮（FAB）— 按住说话按钮已承担此角色
- 不要用渐变色背景
- 不要用大面积彩色
- 不要用卡通头像/AI 形象
- 不要用气泡式聊天 UI（Onboarding 除外）
- 不要在卡片上加阴影
- 不要用 Snackbar/Toast 做重要状态反馈（用卡片状态变化）
- 不要在主屏放"功能入口"或"快捷操作"网格
- 不要加引导弹窗、蒙层教程、红点提示

完整功能清单及优先级分期见 [feature-list.md](feature-list.md)。
